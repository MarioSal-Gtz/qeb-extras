## USUARIOS Y ASIGNADOS

### Logica:
1. Del campo "Vendedor" del CSV, se busca el match en tabla usuario (por nombre fuzzy)
2. El match da el usuario_id y nombre_usuario (creador de la solicitud)
3. Con el usuario_id, se buscan sus equipos (tabla usuario_equipo)
4. De cada equipo, se extraen miembros con roles:
   - Analista de Servicio al Cliente
   - Asesor Analista
   - Disenadores
5. Esos miembros son los "asignados" (con sus IDs y nombres)
6. EL USUARIO TAMBIEN APARECE EN SUS PROPIOS ASIGNADOS (agregado post-migracion)

## NOTAS TECNICAS

### DigitalOcean MySQL 8 vs Hostinger MariaDB:
- MySQL 8 es mas estricto con varchar: rechaza datos mas largos
- Se ampliaron todas las columnas varchar < 255 a varchar(255)
- Campos de texto largo (asignado, descripcion, etc.) a TEXT
- sql_require_primary_key = 0 necesario para importar
- sql_mode debe ser permisivo para fechas (curdate() en datetime)
- SSL requerido: sslcert o sslaccept=accept_invalid_certs

### Oaxaca inventario:
- estado en inventarios = "Oaxaca de Juarez" (correcto, no se cambio)
- plaza = "Oaxaca", municipio = "OAXACA DE JUAREZ"
- El fix fue cambiar backend para buscar por municipio en vez de plaza

### cliente_id en solicitudes:
- El frontend manda CUIC como cliente_id (NO el id real del registro cliente)
- Fix: backend busca cliente por CUIC en vez de por id para nombre correcto
- Afecta: PDF compartir, vista compartir, cliente propuesta page

### FortiClient VPN:
- Servicio: FA_Scheduler (net start/stop FA_Scheduler)
- Necesario para acceder a SAP (10.1.0.9:50000)
- Si no conecta: reiniciar servicio desde CMD admin
